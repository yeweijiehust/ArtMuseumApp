# Packages and Responsibilities

## Prerequisites

- [Repository Tour](../00-orientation/repository-tour.md)
- [Architecture and Data Flow](architecture-and-data-flow.md)

This is a teaching-oriented map of the production code. For a concise lookup table, see [Code Map](../reference/code-map.md).

## Entry Points

### `ArtMuseumApplication.kt`

Business purpose: initialize process-wide dependency injection.

Technical role: `@HiltAndroidApp` makes Hilt generate the application-level object graph.

### `MainActivity.kt`

Business purpose: display the app.

Technical role: Android entry activity, edge-to-edge setup, Compose content host, and theme wrapper.

## Domain Package

### `domain/model/Models.kt`

Defines app-level vocabulary:

- artwork, gallery page, user, language;
- upload and update inputs;
- typed failures.

It has no Android, Retrofit, Room, or Compose imports. This is evidence that the models are framework-independent.

### `domain/repository/Repositories.kt`

Defines four capability boundaries:

- `EndpointRepository`: observe and verify endpoint;
- `PreferencesRepository`: observe and change language;
- `AuthRepository`: session and account actions;
- `GalleryRepository`: reads, refreshes, pagination, and mutations.

### `domain/validation/Validators.kt`

Contains stateless boundary rules shared by forms.

## Data Package

### `data/remote/ArtMuseumApi.kt`

Typed Retrofit declaration for every backend route.

### `data/remote/Dtos.kt`

Serializable network shapes and DTO-to-domain mappers.

### `data/remote/ApiSupport.kt`

Central success/error handling. It translates exceptions, HTTP status, and API error JSON into `AppFailure`.

### `data/remote/SessionCookieJar.kt`

OkHttp cookie adapter for the one authentication cookie. Persists it through `AppPreferences`.

### `data/remote/SessionStore.kt`

In-memory observable current user. This is deliberately separate from cookie persistence.

### `data/local/ImageEntity.kt`

Room storage shape and entity/domain conversion. Adds public and personal list positions.

### `data/local/ImageDao.kt`

SQL access contract generated by Room: observe lists, fetch, upsert, clear positions, and delete.

### `data/local/ArtMuseumDatabase.kt`

Room database declaration and DAO access point.

### `data/preferences/AppPreferences.kt`

DataStore-backed endpoint, language, and cookie persistence. Defines the default endpoint.

### `data/repository/RepositoryImplementations.kt`

The largest coordination file. It implements domain repository contracts and owns cross-source rules:

- verify a replacement endpoint before saving;
- clear session/cache on endpoint changes;
- restore/login/register/logout sessions;
- refresh and merge public/personal cache;
- fall back to cached detail on network failure;
- build multipart upload;
- clear auth after unauthorized protected operations.

## Dependency Injection Package

### `di/AppModule.kt`

Constructs framework objects: JSON parser, OkHttp, Retrofit API, Room database, and DAO.

### `di/RepositoryModule.kt`

Binds domain interfaces to data implementations.

## Presentation Package

### `presentation/ArtMuseumApp.kt`

Owns top-level app state collection, localization provider, scaffold, navigation bars, route graph, login redirects, and destination retention.

### `presentation/viewmodel/ViewModels.kt`

Defines:

- `UiError` and failure mapping;
- immutable UI state classes;
- one ViewModel per major workflow.

### `presentation/ui/Common.kt`

Reusable loading, error, retry, and empty-state rendering.

### `presentation/ui/GalleryScreens.kt`

Public gallery, adaptive grid, cursor trigger, image card, and detail UI.

### `presentation/ui/AuthScreens.kt`

Shared login/register form with local validation.

### `presentation/ui/ManagementScreens.kt`

Photo Picker upload, personal museum, editing, and deletion confirmation.

### `presentation/ui/SettingsScreen.kt`

Endpoint test/save, language selection, account display, and logout.

### `presentation/localization/AppStrings.kt`

Typed English and Simplified Chinese strings plus device-language selection and a CompositionLocal.

### `presentation/theme/Theme.kt`

Material 3 light/dark color schemes based on system theme.

## Resources and Manifests

- `src/main/AndroidManifest.xml`: permission, application, and activity declarations.
- `src/main/res/drawable`: local vector action/navigation icons.
- `src/main/res/xml/network_security_config.xml`: release cleartext denial.
- `src/debug/AndroidManifest.xml`: debug-only network config override.
- `src/debug/res/xml/network_security_config_debug.xml`: local-host HTTP allowance.

## Test Packages

- `data/remote`: API failure mapping and live JSON/OpenAPI compatibility.
- `domain/validation`: rule boundaries.
- `presentation/viewmodel`: observable behavior with fake repositories.
- `presentation/ui`: localized prompt and screenshot behavior.
- `androidTest`: real Android identity and protected navigation behavior.

Read [Testing and Continuous Integration](../06-quality/testing-and-ci.md) for how the test types differ.

## A Useful Reading Heuristic

When changing behavior, read in this order:

1. domain model and repository contract;
2. relevant UI state and ViewModel;
3. repository implementation;
4. source adapter such as API or DAO;
5. Compose screen;
6. tests.

That order reveals intent before framework mechanics.
